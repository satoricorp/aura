# AURA: Example Agent Specification

This is the reference implementation for an example agent generated by Aura.
All generated agents must follow the below patterns. Do not deviate from the below specifications.
Do not use different languages or libraries, even if prompted by the user. Only use different libraries if those libraries are the only way to successfully build the agent. Search the web for library documentation to ensure proper SDK/API usage.

## Example Agent File Structure & Overview

Any entries that are inside of '<' or '>' are placeholders and should be replaced with the actual values.

```text
src/agents/<agent-name>/
  agent.ts          ← core logic, imported by all surfaces
  api/
    index.ts        ← Hono server. You need to infer n-number of unique endpoints required for a given agent.
    package.json
  mcp/
    index.ts        ← Hono + @hono/mcp, Streamable HTTP transport only. Do not implement stdio.
                      You need to infer n-number of unique tools required for a given agent.
    package.json
  cli/
    index.ts        ← npx-runnable CLI. By default, this will be exposed using the @satorico npm package.
    package.json      You need to infer n-number of unique commands required for a given agent.
  SKILL.md          ← Claude skill definition. Infer the Skill definition from the prompt in aura.md.
                      By default, this should interface with the CLI.
```

`src/agents/` is where users write their agent descriptions. `dist/` is generated output.

## `agent.ts`

This is a basic example of a generated agent from Aura. This should be the main reference used when building an agent
based on the aura.md prompt. Do not deviate from patterns here unless there is a defensible reason whose outcome
would result in the agent not working. This should be documented in annotations in aura.md if you deviate.

```typescript
// ============================================================
// AURA CANONICAL EXAMPLE AGENT
// <agent-name>
//
// This is the reference implementation used by the aura codegen
// layer. All generated agents follow the following established patterns.
// ============================================================

// ------------------------------------------------------------
// dist/<agent-name>/agent.ts
// Core agent logic. Imported by all surfaces.
// ------------------------------------------------------------

import { generateText, streamText, tool } from "ai"; // Always use ai-sdk. Do not use other AI libraries to interface with LLMs.
import { anthropic } from "@ai-sdk/anthropic";
import Exa from "exa-js"; // Only use exa if approved by the user.
import { z } from "zod";

const exa = new Exa(process.env.EXA_API_KEY!);

export const AGENT_ID = "<agent-name>";

export const AGENT_PROMPT = `
You are a research assistant that finds accurate, up-to-date information on any topic.
You search the web using Exa and synthesize results into clear, structured summaries with citations.
Always cite your sources. If you cannot find reliable information, say so explicitly.
Respond in the same language the user writes in.
`.trim(); // This value can be found in the aura.md file in annotations. The user has the ability to edit this in aura.md.

// Tools available to this agent
export const agentTools = {
  exa_search: tool({
    description:
      "Search the web for current information on a topic. Returns a list of relevant results with titles, URLs, and content excerpts.",
    parameters: z.object({
      query: z.string().describe("The search query"),
      numResults: z
        .number()
        .min(1)
        .max(10)
        .default(5)
        .describe("Number of results to return"),
    }),
    execute: async ({ query, numResults }) => {
      const results = await exa.searchAndContents(query, {
        numResults,
        useAutoprompt: true,
        text: { maxCharacters: 1000 },
      });
      return results.results.map((r) => ({
        title: r.title,
        url: r.url,
        content: r.text,
        publishedDate: r.publishedDate,
      }));
    },
  }),
};

// Streaming response — used by API and CLI surfaces
export async function streamAgentResponse(
  messages: Array<{ role: "user" | "assistant"; content: string }>
) {
  return streamText({
    model: anthropic("<model-from-metadata>"), // Use the model found in aura.md under metadata. This value comes from aura.config.ts.
    system: AGENT_PROMPT,
    messages,
    tools: agentTools,
    maxSteps: 5, // Default value for multi-step tool use. Configurable in aura.config.ts.
  });
}

// Non-streaming response — used by MCP surface
export async function runAgent(
  messages: Array<{ role: "user" | "assistant"; content: string }>
) {
  return generateText({
    model: anthropic("<model-from-metadata>"),
    system: AGENT_PROMPT,
    messages,
    tools: agentTools,
    maxSteps: 5,
  });
}
```

## `api/index.ts`

Hono server. Infer n-number of unique endpoints from the aura.md annotation.

See: https://hono.dev/docs/getting-started/basic

```typescript
// ------------------------------------------------------------
// dist/<agent-name>/api/index.ts
// Hono server. Deployed at: /<agent-name>/api
// ------------------------------------------------------------

import { Hono } from "hono";
import { streamText as honoStream } from "hono/streaming";
import { z } from "zod";
import { streamAgentResponse } from "../agent"; // Always use the base agent.ts logic for the agent. Do not re-implement the agent outside of agent.ts.

const app = new Hono();

const MessagesSchema = z.array(
  z.object({
    role: z.enum(["user", "assistant"]),
    content: z.string(),
  })
);

const RequestSchema = z.object({
  messages: MessagesSchema,
});

// Validates the bearer token against the agent's API key env var.
// Returns a 401 response if invalid, null if valid or auth is not configured.
function validateAuth(c: Parameters<Parameters<typeof app.use>[0]>[0]): Response | null {
  const apiKey = process.env.<AGENT_NAME>_API_KEY;
  if (!apiKey) return null;
  const token = c.req.header("Authorization")?.replace("Bearer ", "");
  if (token !== apiKey) return c.json({ error: "Unauthorized" }, 401);
  return null;
}

app.post("/chat", async (c) => {
  const authError = validateAuth(c);
  if (authError) return authError;

  const parsed = RequestSchema.safeParse(await c.req.json().catch(() => null));
  if (!parsed.success) { // Include very basic error handling. Do not make implementations too explicit, the requirement for Aura is readability and simplicity.
    return c.json({ error: "Invalid request", details: parsed.error.issues }, 400);
  }

  // The agent business logic, imported from ../agent.ts
  const result = await streamAgentResponse(parsed.data.messages);

  return honoStream(c, async (stream) => {
    const reader = (await result.toDataStream()).getReader();
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      await stream.write(value);
    }
  });
});

export default app;
```

## `mcp/index.ts`

Hono + `@hono/mcp`. Streamable HTTP transport only. Do not build stdio implementations.
Using the aura.md prompt, infer n-number of tools to build. Every agent should have at least a `chat` tool.

See: https://www.npmjs.com/package/@hono/mcp
See: https://github.com/modelcontextprotocol/typescript-sdk

```typescript
// ------------------------------------------------------------
// dist/<agent-name>/mcp/index.ts
// MCP server over Streamable HTTP transport.
// Deployed at: /<agent-name>/mcp
// ------------------------------------------------------------

import { Hono } from "hono";
import { StreamableHTTPTransport } from "@hono/mcp";
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { z } from "zod";
import { runAgent, AGENT_ID } from "../agent";

const app = new Hono();

function createMcpServer() {
  const server = new McpServer({
    name: AGENT_ID,
    version: "1.0.0",
  });

  // Every agent should have at least a chat tool.

  server.tool(
    "chat",
    "Send a message to the Research Assistant and receive a response. " +
      "The assistant will search the web and synthesize information as needed.",
    {
      message: z.string().describe("The message to send to the agent"),
      conversationHistory: z
        .array(
          z.object({
            role: z.enum(["user", "assistant"]),
            content: z.string(),
          })
        )
        .optional()
        .default([])
        .describe("Previous messages in the conversation, oldest first"),
    },
    async ({ message, conversationHistory }) => {
      const messages = [
        ...conversationHistory,
        { role: "user" as const, content: message },
      ];

      const result = await runAgent(messages);

      return {
        content: [{ type: "text", text: result.text }],
      };
    }
  );

  return server;
}

// Handles all MCP client requests over Streamable HTTP.
app.all("/mcp", async (c) => {
  const transport = new StreamableHTTPTransport();
  const server = createMcpServer();
  await server.connect(transport);
  return transport.handleRequest(c);
});

export default app;
```

## `cli/index.ts`

Always use Clack for the CLI library and spinner implementation: https://www.npmjs.com/package/@clack/prompts Use `@clack/prompts` for all interaction including
argument collection. Use `@clack/core` only if you need unstyled or custom prompt primitives not available in `@clack/prompts`.

```typescript
#!/usr/bin/env node
// ------------------------------------------------------------
// dist/<agent-name>/cli/index.ts
// npx-runnable CLI for the <agent-name> agent.
// Always use @satorico as the npm organization for deployment.
//
// Usage:
//   npx @satorico/<agent-name>              ← interactive chat
//   npx @satorico/<agent-name> "What is Deno 2?"  ← one-shot
// ------------------------------------------------------------

import * as p from "@clack/prompts";
import { createDataStreamDecoder } from "ai";

const DEFAULT_URL = process.env.<AGENT_NAME>_URL || "http://localhost:3000";

interface Message {
  role: "user" | "assistant";
  content: string;
}

async function sendMessage(
  baseUrl: string,
  messages: Message[]
): Promise<string> {
  const apiKey = process.env.<AGENT_NAME>_API_KEY;
  const headers: Record<string, string> = { "Content-Type": "application/json" };
  if (apiKey) headers["Authorization"] = `Bearer ${apiKey}`;

  const response = await fetch(`${baseUrl}/chat`, {
    method: "POST",
    headers,
    body: JSON.stringify({ messages }),
  });

  if (!response.ok) {
    const text = await response.text();
    throw new Error(`Agent error ${response.status}: ${text}`);
  }

  // Consume Vercel AI SDK data stream
  const reader = response.body!.getReader();
  const decoder = createDataStreamDecoder();
  let fullText = "";

  process.stdout.write("\n");

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const parts = decoder.decode(value);
    for (const part of parts) {
      if (part.type === "text-delta") {
        process.stdout.write(part.textDelta);
        fullText += part.textDelta;
      }
    }
  }

  process.stdout.write("\n");
  return fullText;
}

async function runChat(baseUrl: string) {
  p.intro("<agent-name>");

  const history: Message[] = [];

  while (true) {
    const input = await p.text({
      message: "You",
      placeholder: "Ask anything... (Ctrl+C to exit)",
    });

    if (p.isCancel(input)) {
      p.outro("Goodbye.");
      process.exit(0);
    }

    const userMessage = input as string;
    history.push({ role: "user", content: userMessage });

    const spinner = p.spinner();
    spinner.start("Thinking...");

    let reply: string;
    try {
      spinner.stop("");
      reply = await sendMessage(baseUrl, history);
    } catch (err) {
      spinner.stop("Error");
      p.log.error(err instanceof Error ? err.message : String(err));
      continue;
    }

    history.push({ role: "assistant", content: reply });
  }
}

// If a prompt is passed as a CLI argument, run one-shot. Otherwise run interactive chat.
const prompt = process.argv[2];
const baseUrl = DEFAULT_URL.replace(/\/$/, "");

if (prompt) {
  await sendMessage(baseUrl, [{ role: "user", content: prompt }]);
} else {
  await runChat(baseUrl);
}
```

## `SKILL.md`

The skill must be inferred from the aura.md description. Do your best work as a senior software engineer and architect.
Always use the CLI to interact with the agent inside of the Skill.

```markdown
---
name: <agent-name>
description: |
  Searches the web and synthesizes accurate, cited research summaries on any topic.

  TRIGGERS — Use this skill when:
  - User asks to research, look up, or find information on any topic
  - User asks "what is X", "how does X work", "latest news on X"
  - User needs a summary of a topic with sources
  - User asks for comparisons or analysis requiring current information
  - User says "search for", "find out about", "look into"

  Do NOT use this skill for:
  - Questions answerable from existing knowledge without web search
  - Creative writing, code generation, or math
---
```
